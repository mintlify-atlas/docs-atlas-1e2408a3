This guide walks you through creating and running a minimal Sinatra application. You will have a working web server responding to HTTP requests in under five minutes.

<Steps>
  <Step title="Install the gems">
    You need Sinatra, a Rack-compatible server (Puma), and the `rackup` command-line tool. Install all three with a single command:

    ```bash
    gem install sinatra rackup puma
    ```

    - **sinatra** — the framework itself
    - **puma** — a fast, concurrent web server
    - **rackup** — the CLI tool for starting Rack applications
  </Step>

  <Step title="Create your app file">
    Create a file called `myapp.rb` with the following content:

    ```ruby myapp.rb
    require 'sinatra'

    # Respond to GET /
    get '/' do
      'Hello world!'
    end

    # Respond to GET /hello/:name with a named parameter
    get '/hello/:name' do
      # params['name'] captures the value from the URL segment
      "Hello, #{params['name']}!"
    end
    ```

    Routes in Sinatra are HTTP method and URL pattern pairs. Each route is a block that returns the response body as a string. Named parameters like `:name` are accessible via `params['name']`.

    You can also access named parameters directly as block arguments:

    ```ruby
    get '/hello/:name' do |name|
      "Hello, #{name}!"
    end
    ```
  </Step>

  <Step title="Run the app">
    Start the server by running the file with Ruby:

    ```bash
    ruby myapp.rb
    ```

    You should see output similar to:

    ```
    == Sinatra (v4.2.1) has taken the stage on 4567 for development with backup from Puma
    Puma starting in single mode...
    * Listening on http://0.0.0.0:4567
    ```

    <Note>
      Code changes do not take effect until you restart the server. Stop the process with `Ctrl+C` and run `ruby myapp.rb` again after making edits.
    </Note>
  </Step>

  <Step title="Open your app in a browser">
    Visit the following URLs to see your routes in action:

    | URL | Response |
    |-----|----------|
    | `http://localhost:4567/` | `Hello world!` |
    | `http://localhost:4567/hello/alice` | `Hello, alice!` |
    | `http://localhost:4567/hello/world` | `Hello, world!` |

    You can also test with `curl`:

    ```bash
    curl http://localhost:4567/hello/alice
    # => Hello, alice!
    ```
  </Step>
</Steps>

<Tip>
  Restarting the server on every change gets tedious quickly. Use [rerun](https://github.com/alexch/rerun) or [rack-unreloader](https://github.com/jeremyevans/rack-unreloader) to automatically reload your app when files change.

  ```bash
  gem install rerun
  rerun ruby myapp.rb
  ```
</Tip>

## What's next

Now that you have a working app, explore the rest of the framework:

- Add more HTTP verbs: `post`, `put`, `patch`, `delete`
- Render templates with `erb :index` or `haml :index`
- Use `before` and `after` filters to run code around every request
- Switch to modular style with `Sinatra::Base` for larger applications

See [Routes](/guides/routes) for a full reference on route patterns, conditions, and return values.
