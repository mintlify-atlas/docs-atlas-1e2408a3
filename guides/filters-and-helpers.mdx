## Before filters

Before filters run before each request, in the same context as routes. They can read and modify both the request and the response. Instance variables set in a before filter are accessible in the matching route and in templates.

```ruby
before do
  @note = 'Hi!'
  request.path_info = '/foo/bar/baz'
end

get '/foo/*' do
  @note         #=> 'Hi!'
  params['splat'] #=> 'bar/baz'
end
```

## After filters

After filters run after each request. They share the same context as routes, so they can access instance variables set in before filters and routes.

```ruby
after do
  puts response.status
end
```

<Note>
Unless you use the `body` helper to set the response body explicitly, the body is not yet available in an after filter — it is generated later by the Rack handler.
</Note>

## Pattern-matching filters

Filters accept an optional URL pattern. The filter only runs when the request path matches:

```ruby
before '/protected/*' do
  authenticate!
end

after '/create/:slug' do |slug|
  session[:last_slug] = slug
end
```

## Conditions in filters

Filters support the same conditions as routes:

```ruby
before :agent => /Songbird/ do
  # runs only for Songbird user-agent requests
end

after '/blog/*', :host_name => 'example.com' do
  # runs only for matching path on example.com
end
```

## Helpers

Use the top-level `helpers` method to define methods available in routes and templates:

```ruby
helpers do
  def bar(name)
    "#{name}bar"
  end
end

get '/:name' do
  bar(params['name'])
end
```

You can also include one or more modules:

```ruby
module FooUtils
  def foo(name) "#{name}foo" end
end

module BarUtils
  def bar(name) "#{name}bar" end
end

helpers FooUtils, BarUtils
```

## Built-in helper methods

### `session`

Enable sessions and access the session hash:

```ruby
enable :sessions

get '/' do
  "value = " << session[:value].inspect
end

get '/:value' do
  session['value'] = params['value']
end
```

### `halt`

Stop a request immediately and optionally set status, headers, and body:

```ruby
halt                            # empty 200
halt 410                        # status only
halt 'this will be the body'    # body only
halt 401, 'go away!'            # status + body
halt 402, { 'Content-Type' => 'text/plain' }, 'revenge'
halt erb(:error)                # render a template and halt
```

### `pass`

Abandon the current route block and hand control to the next matching route. Returns a 404 if no subsequent route matches:

```ruby
get '/guess/:who' do
  pass unless params['who'] == 'Frank'
  'You got me!'
end

get '/guess/*' do
  'You missed!'
end
```

### `redirect`

Trigger a browser redirect. Uses `to` (aliased from `url`) to build the target URL:

```ruby
get '/foo' do
  redirect to('/bar')
end
```

Pass a status code or message as additional arguments:

```ruby
redirect to('/bar'), 303
redirect 'http://www.google.com/', 'wrong place, buddy'
```

Redirect back to the referring page:

```ruby
get '/bar' do
  do_something
  redirect back
end
```

### `body`, `status`, `headers`

Set or get response components at any point in the execution flow:

```ruby
get '/foo' do
  status 418
  headers \
    "Allow"   => "BREW, POST, GET, PROPFIND, WHEN",
    "Refresh" => "Refresh: 20; https://ietf.org/rfc/rfc2324.txt"
  body "I'm a teapot!"
end

after do
  puts body    # accessible in after filter when set via body helper
end
```

### `stream`

Send a streaming response without blocking the thread:

```ruby
get '/' do
  stream do |out|
    out << "It's gonna be legen -\n"
    sleep 0.5
    out << " (wait for it) \n"
    sleep 1
    out << "- dary!\n"
  end
end
```

Set the optional parameter to `:keep_open` to keep the stream open for later writes (useful for Server-Sent Events):

```ruby
get '/sse' do
  stream(:keep_open) do |out|
    connections << out
  end
end
```

<Note>
Streaming behaviour depends on the web server. Some servers buffer the full body before sending. Streaming does not work with Shotgun. Check `out.closed?` before writing to avoid errors when the client disconnects.
</Note>

### `logger`

Access a `Logger` instance wired to your Rack handler's logging settings:

```ruby
get '/' do
  logger.info "loading data"
end
```

If logging is disabled, `logger` returns a dummy object so your code does not need to guard against `nil`.

<Note>
Logging is only enabled by default for `Sinatra::Application`. When subclassing `Sinatra::Base`, enable it explicitly:

```ruby
class MyApp < Sinatra::Base
  configure :production, :development do
    enable :logging
  end
end
```
</Note>

### `mime_type` and `content_type`

Register a MIME type by file extension, then use it with `content_type`:

```ruby
configure do
  mime_type :foo, 'text/foo'
end

get '/' do
  content_type :foo
  "foo foo foo"
end
```

### `url` / `to`

Generate absolute URLs that are aware of reverse proxies and Rack routers:

```ruby
# in a Haml template
%a{ :href => url('/foo') } foo

# redirect using the to alias
redirect to('/bar')
```

### `cache_control`, `expires`, `etag`, `last_modified`

Set HTTP caching headers. Call `etag` or `last_modified` early — they will flush a 304 immediately if the client already has a fresh copy:

```ruby
get "/article/:id" do
  @article = Article.find params['id']
  last_modified @article.updated_at
  etag @article.sha1
  erb :article
end
```

Set `Cache-Control` directly:

```ruby
get '/' do
  cache_control :public
  "cache it!"
end
```

<Tip>
Set caching headers in a `before` filter to apply them across multiple routes:

```ruby
before do
  cache_control :public, :must_revalidate, :max_age => 60
end
```
</Tip>

Use `expires` to set `Cache-Control` and `Expires` together:

```ruby
before do
  expires 500, :public, :must_revalidate
end
```

Use a weak ETag when exact byte-for-byte validation is not required:

```ruby
etag @article.sha1, :weak
```

### `send_file`

Return the contents of a file as the response:

```ruby
get '/' do
  send_file 'foo.png'
end

get '/download' do
  send_file 'foo.png', :type => :jpg
end
```

Available options:

| Option | Description |
|---|---|
| `:filename` | File name sent in the response. Defaults to the real file name. |
| `:last_modified` | Value for the `Last-Modified` header. Defaults to the file's `mtime`. |
| `:type` | Value for `Content-Type`. Guessed from the extension if absent. |
| `:disposition` | `Content-Disposition` value: `nil` (default), `:attachment`, or `:inline`. |
| `:length` | Value for `Content-Length`. Defaults to the file size. |
| `:status` | Status code. Useful when serving a static file as an error page. |

Sinatra handles range requests automatically when `send_file` is used.

### `attachment`

Tell the browser to store the response on disk rather than display it:

```ruby
get '/' do
  attachment
  "store it!"
end

get '/report' do
  attachment "report.txt"
  "store it!"
end
```

### `request`

Access the full Rack request object from any filter, route, or error handler:

```ruby
# app running on http://example.com/example
get '/foo' do
  request.accept              # ['text/html', '*/*']
  request.accept? 'text/xml'  # true
  request.body                # request body (IO or StringIO)
  request.scheme              # "http"
  request.script_name         # "/example"
  request.path_info           # "/foo"
  request.port                # 80
  request.request_method      # "GET"
  request.query_string        # ""
  request.content_length      # length of request.body
  request.media_type          # media type of request.body
  request.host                # "example.com"
  request.get?                # true (similar methods for other verbs)
  request.form_data?          # false
  request["some_param"]       # shortcut to the params hash
  request.referrer            # referrer URL or '/'
  request.user_agent          # used by the :agent condition
  request.cookies             # hash of browser cookies
  request.xhr?                # true if Ajax request
  request.url                 # "http://example.com/example/foo"
  request.path                # "/example/foo"
  request.ip                  # client IP address
  request.secure?             # false (true over SSL)
  request.forwarded?          # true if behind a reverse proxy
  request.env                 # raw Rack env hash
end
```

Some properties like `path_info` and `script_name` are writable:

```ruby
before { request.path_info = "/" }

get "/" do
  "all requests end up here"
end
```

Parse a JSON request body:

```ruby
post "/api" do
  request.body.rewind
  data = JSON.parse request.body.read
  "Hello #{data['name']}!"
end
```

### `time_for`

Convert a value to a `Time` object. Understands `DateTime`, `Date`, strings, and integers:

```ruby
get '/' do
  pass if Time.now > time_for('Dec 23, 2016')
  "still time"
end
```

Override it to add custom shorthands used by `expires`, `last_modified`, and related helpers:

```ruby
helpers do
  def time_for(value)
    case value
    when :yesterday then Time.now - 24*60*60
    when :tomorrow  then Time.now + 24*60*60
    else super
    end
  end
end

get '/' do
  last_modified :yesterday
  expires :tomorrow
  "hello"
end
```

### `find_template`

Used internally to locate template files. Override it to implement custom lookup logic:

```ruby
set :views, ['views', 'templates']

helpers do
  def find_template(views, name, engine, &block)
    Array(views).each { |v| super(v, name, engine, &block) }
  end
end
```

You can also call it directly to inspect candidate paths:

```ruby
find_template settings.views, 'foo', Tilt[:haml] do |file|
  puts "could be #{file}"
end
```
